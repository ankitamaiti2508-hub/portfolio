Agent 1 · Retailer Verification
1 🔐 Verification Agent
Confirms retailer identity before proceeding with the call.
Greets retailer and asks: "Am I speaking with Store X?"
Cross-checks retailer response against Retailer DB (store name, owner name, phone)
Confirms identity match before passing context to next agent
✓ Verified → proceed ✗ Wrong store → Close Call Agent
Retailer DB lookup Identity matching
Verified · retailer context loaded
Agent 2 · Introduction & Context Setting
2 💬 Intro Agent
Sets the context for the call, builds rapport before pitching.
Explains reason for call: "Your salesman is absent today, so I'm calling to take your order."
Opens with a rapport question: "How is business going? How are sales this week?"
Reads retailer response tone — positive / neutral / busy / complaint — and flags to orchestrator
✓ Engaged → Pitch Agent Busy → Callback Agent Complaint → Service Agent
Retailer engaged · tone assessed
Agent 3 · Product & Scheme Pitch
3 🛒 Pitch Agent
Core selling agent — personalised pitch using past order data and active schemes.
Past order lookup: Identifies top SKUs the retailer orders regularly and leads with those
Scheme surfacing: Surfaces the most attractive active schemes and offers relevant to this retailer's category
Stock check: Asks retailer about current stock levels to determine pitch strategy
Has stock gaps → Stock Objection Agent Full stock → Low AOV Pitch Agent
Past orders DB Schemes & offers DB SKU catalogue
Stock status known · pitch strategy selected
Agent 4A · Stock Objection
4 📦 Stock Agent
Retailer says they have stock — find gaps and pitch accordingly.
Asks which SKUs they have sufficient stock of
Cross-references against their typical order basket to identify missing SKUs
Pitches specifically on the gaps: "I see you usually order X — are you running low on that?"
Order history Inventory data
OR
Agent 4B · Full Stock / Low AOV
4 💡 AOV Agent
Retailer has full stock — pivot to low-AOV products and alternate strategies.
Pivots to low-AOV / impulse SKUs: small packs, new launches, seasonal items
Tries alternate pitch angles: scheme urgency ("offer ends this week"), social proof ("neighbouring stores are picking this up")
If no traction — gracefully moves to order confirmation or close
SKU catalogue Schemes DB
Order basket confirmed
Agent 5 · Order Confirmation
5 ✅ Order Agent
Reads back the full order, confirms quantities, and places it in the OMS.
Summarises the order: SKUs, quantities, applicable schemes
Gets explicit confirmation from retailer
Places order in Order Management System · shares order reference
Order Management System Scheme validation
Order placed · closing
Agent 6 · Call Closing
6 🤝 Close Agent
Closes the call warmly and logs outcome.
Thanks retailer · shares order summary and expected delivery
Logs call outcome, order ID, and any flags to CRM
Ends call gracefully
CRM / ERP Call logging
⚡ Interrupt Agents
Triggered at any point · Resolve & return to flow · 5 agents
🛡️ Skepticism Handler Agent
Triggered when retailer doubts who is calling — "Who are you? Where is this call from?"

· Proactively shares trust-building details the retailer recognises:
  — Distributor name they order from
  — Last order date and SKUs
  — Delivery address on file
· Familiarity with their account builds credibility fast
· Returns to verification or pitch flow once trust is established
Retailer DB Distributor mapping Order history
❌ Wrong Store Agent
Triggered during Verification if retailer identity doesn't match.

· Apologises for the inconvenience
· Closes call gracefully
· Logs as wrong number in CRM
🕐 Callback Agent
Triggered if retailer says they are busy.

· Acknowledges: "No problem, when would be a good time?"
· Finds a mutually suitable time
· Schedules callback in system
· Closes call politely
🎧 Service Request Agent
Triggered if retailer raises a complaint (e.g. cooler issues, delivery problem).

· Acknowledges the issue empathetically
· Refers to Service FAQs DB for resolution guidance
· Raises a service ticket
· Returns to pitch flow after resolution
Service FAQs DB Ticketing System
❗ Objection Handler Agent

· Uses pre-trained objection handling scripts
· Surfaces relevant scheme to counter
· Returns to pitch flow
